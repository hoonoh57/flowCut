FlowCut 전체 기능 해설 (2026-04-22 기준)
기본 NLE 에디터 (Phase 1-2)
FlowCut의 토대입니다. 브라우저에서 동작하는 멀티트랙 비디오 에디터로, 비디오/이미지/오디오/텍스트 4종 트랙을 지원합니다. 미디어 파일을 드래그하면 타임라인에 클립이 생성되고, 클립의 이동/리사이즈/분할/삭제가 모두 가능합니다. 모든 편집 작업은 Command 패턴으로 구현되어 Ctrl+Z/Y로 무제한 Undo/Redo됩니다. 프리뷰 캔버스에서 실시간 렌더링을 보면서 편집할 수 있고, 16:9/9:16/1:1/4:3/4:5/21:9 비율을 한 클릭으로 전환합니다. FFmpeg를 통해 MP4/WebM/GIF로 내보내기하며, 텍스트 클립은 30종 이상의 프리셋 스타일(애니메이션 포함)을 제공합니다.

AI 영상 생성 파이프라인 (Phase 3.1-3.3)
프롬프트를 입력하면 ComfyUI를 통해 이미지를 생성하고, 그 이미지를 Wan2.2 모델로 5초(81프레임) 비디오로 변환합니다. 생성된 비디오는 자동으로 타임라인에 배치됩니다. Ollama(로컬 LLM)가 프롬프트를 분석하여 씬을 구성하고, VideoDirector가 Hook→Context→Payoff→Loop 구조의 beat 템플릿으로 15씬 영상을 자동 구성합니다. 이 모든 과정이 "프롬프트 하나"로 시작됩니다.

코딩 표준 + 하드코딩 제거 (Phase 3.4)
프로젝트 크기(1920x1080 등)를 코드에 직접 쓰지 않고 DEFAULT_PROJECT 상수에서 가져오는 규칙을 수립했습니다. 10개 파일의 하드코딩 위반을 수정하고, 값 우선순위 체인(스크립트 지정값 → editorStore → DEFAULT_PROJECT)을 확립했습니다. .ai-rules.json에 개발 환경 규칙을 명문화하여 PowerShell/Node.js 환경에서의 반복 에러를 방지합니다.

FlowScript 액션 확장 (Phase 3.5)
AI가 편집을 자동화하기 위한 스크립트 인프라입니다. 기존 14개 액션에서 27개로 확장하여 커버리지 52%에서 90%+로 끌어올렸습니다. 추가된 핵심 액션은 addClip(클립 직접 추가, 트랙 자동 생성 포함), setClipProperty(개별 속성 변경), addTrack/removeTrack, setProject(프로젝트 설정 변경), trim, duplicate입니다. 모든 액션은 브라우저 Console의 __flowcut.executeScript()로 테스트 검증되었습니다.

TTS 오디오 파이프라인 (Phase 3.5)
Edge TTS를 연동하여 텍스트를 음성으로 변환합니다. 한국어/영어/일본어/중국어 등 다국어를 지원하며, Windows 환경에서 한글 인코딩이 깨지는 문제를 파일 기반 입력(--file)으로 해결했습니다. generateTTS 액션으로 FlowScript에서 자동 호출 가능하고, 생성된 오디오는 미디어 패널에 자동 등록됩니다. 캐릭터별 다른 보이스 프리셋을 지정할 수 있어, 유나는 SunHi 보이스, 민호는 InJoon 보이스처럼 구분됩니다.

I2V 클립 길이 연장 + 업스케일 (Phase 3.5)
Image-to-Video 생성 프레임 수를 33프레임(2초)에서 81프레임(5초)으로 늘렸습니다. video-i2v.json 워크플로우와 ScriptEngine/server.cjs 모두 동기화되어 있습니다. 업스케일 엔드포인트(/api/comfyui/upscale)를 추가하여 480p 생성물을 FFmpeg Lanczos 필터로 확대할 수 있고, upscale 액션으로 FlowScript에서 호출 가능합니다.

크로스페이드 전환 + 타임라인 강화 (Phase 3.6)
FFmpeg xfade 필터를 위한 buildXfadeFilter 함수를 추가하고, VideoDirector가 beat 사이에 dissolve/fade 전환 정보를 FlowScript에 포함시킵니다. 타임라인 클립에 비디오(🎬)/이미지(🖼) 타입 아이콘이 표시되고, fadeIn/fadeOut이 있는 클립에 FI/FO 레이블이 나타납니다. 프리뷰 캔버스의 비디오 동기화를 개선하여, 정지 시 정확한 프레임 seek와 재생 시 drift correction(0.15초 임계값)이 동작합니다.

버그 수정 (Phase 3.6.1)
텍스트 클립 추가 시 텍스트 트랙이 없으면 자동 생성하도록 addClip 액션을 수정했습니다. Space bar로 재생/정지하는 togglePlayback 함수가 playbackSlice에 누락되어 모든 키보드 단축키가 먹통이 되는 문제를 해결했습니다. 재생 시 프레임이 한번에 여러 프레임을 건너뛰어 커서가 끊기는 문제를 1프레임씩 정확히 진행하도록 수정했습니다.

In-Out Range 작업 영역 (Phase 3.7)
프로 NLE의 필수 기능인 작업 영역을 구현했습니다. I키로 시작점, O키로 끝점을 설정하면 타임라인 Ruler에 파란색 구간이 표시됩니다. 재생 시 Out point에서 자동 정지하며, 루프 모드를 켜면 In point로 돌아가 반복 재생합니다. Export 패널에서 In-Out 구간이 설정되어 있으면 해당 구간만 내보냅니다. Esc키로 구간을 해제합니다.

Open Architecture — Provider Interface (P2 Week 1)
FlowCut이 ComfyUI, 특정 GPU, 특정 클라우드에 종속되지 않도록 하는 추상화 계층입니다. FlowCutProvider 인터페이스를 정의하고, ProviderManager가 이미지/비디오/TTS/보간/업스케일/LLM 6개 영역의 활성 Provider를 관리합니다. 사용자는 flowcut.providers.json에서 "active": "comfyui"를 "active": "fal"로 바꾸기만 하면 코드 변경 없이 클라우드 API로 전환됩니다. 체이닝 전략도 Provider 능력에 따라 S~D 등급으로 자동 선택됩니다.

World Context — 영상 세계관 일관성 (P2 Week 1)
FlowCut의 핵심 차별화 기능입니다. FlowScript에 world 섹션을 추가하여 캐릭터(외모+복장+변형), 장소(설명+소품+조명), 소품, 시대, 장르, 비주얼 스타일을 한번 정의하면 모든 씬 생성에 자동 주입됩니다. 경쟁 제품(Runway, Kling)이 "캐릭터 얼굴" 한 가지에만 집중하는 반면, FlowCut은 복장 변형, 장소 소품, 시대 장르, 스타일 오버라이드까지 영상의 전체 세계관을 구조적으로 관리합니다.

검증 결과: 사용자가 "reading letter at window" 7단어만 입력하면, World Context가 캐릭터/복장/장소/스타일을 자동 주입하여 40+ 단어의 상세 프롬프트로 확장합니다. 동일 시드(847291036)로 여러 씬을 생성하면 동일 인물이 유지됩니다.

캐릭터 레지스트리 (P2 Week 2)
캐릭터를 시드 번호 + 프롬프트와 함께 DB에 등록하고, 키 하나("yuna")로 언제든 호출하는 시스템입니다. 서버 API(/api/registry/character)로 등록/조회/삭제가 가능하고, localStorage에도 저장됩니다. World Context에서 "$ref": "registry:yuna"로 참조하면 등록된 캐릭터 정보가 자동 로드됩니다. "$seed" 단축 표기도 지원하여 시드 번호만으로 캐릭터를 지정할 수 있습니다.

시네마틱 프롬프트 강화 (P2 Week 2)
42가지 카메라 무브먼트(Dolly/Pan/Tilt/Tracking/Crane/Orbit/Zoom/Steadicam/Handheld/Aerial/Static/Special/Hyperlapse/Cinematic)가 데이터베이스화되어 있습니다. 씬 유형을 자동 감지(portrait/landscape/action/dialogue/emotion/walk/food/product/epic)하고, 에너지 레벨(low/medium/high)에 따라 적절한 카메라 움직임, 조명, 샷 사이즈를 자동 선택합니다. LLM 강화 모드(enhanceWithLLM)도 지원하여 Ollama/OpenAI가 씬을 분석하고 시네마틱 키워드를 추가합니다.

검증 결과: 프롬프트에 자동으로 "medium close-up, chest up, orbiting counter-clockwise around subject, Rembrandt lighting, dramatic side light, 8K, highly detailed, professional photography"가 추가됩니다.

현재 FlowCut은 기술 데모 수준이고, 품질 좋은 작품을 만들려면 아직 상당한 거리가 있습니다.

현재 상태 (100점 만점 기준: 25점)

지금 할 수 있는 것은 AI 이미지 생성 후 5초 저해상도 비디오 변환, 씬 간 체인 연결 (불완전), 기본 BGM 삽입 (ACE-Step), TTS 음성 생성, 그리고 타임라인 편집 + export입니다.

품질 작품에 필요한 것과 현실적 격차

**영상 품질 (현재 → 목표)**으로 보면, 지금은 480×832 해상도에 5초 클립이고 업스케일해도 여전히 뭉개집니다. 목표는 최소 720p, 이상적으로 1080p에 10-15초 클립입니다. Wan2.2는 구조적으로 해상도와 길이에 한계가 있어서, HunyuanVideo나 CogVideoX 같은 더 나은 모델이 필요하거나, 아니면 VRAM 24GB 이상의 GPU가 필요합니다. 현재 GPU가 뭔지에 따라 이 부분은 하드웨어 벽일 수 있습니다.

**씬 연속성 (현재 → 목표)**에서는, 체인이 작동하지만 Wan2.2 i2v가 시작 이미지를 "참고"할 뿐 정확히 이어가지 않습니다. 인물이 등장하면 얼굴, 의상, 체형이 씬마다 달라집니다. IPAdapter나 ControlNet을 조합해도 완벽한 연속성은 현재 오픈소스 모델로는 달성하기 어렵습니다.

**오디오 (현재 → 목표)**를 보면, ACE-Step BGM은 "음악 비슷한 것"이지 실제 음악이 아닙니다. TTS는 작동하지만 나레이션 타이밍이 영상과 수동 동기화입니다. 립싱크는 아예 없습니다. BGM은 로열티프리 라이브러리를 쓰는 게 현실적이고, TTS 자동 싱크와 SadTalker/MuseTalk 같은 립싱크 모델 통합이 필요합니다.

**편집 완성도 (현재 → 목표)**에서, 크로스페이드를 방금 추가했지만 export에서만 적용되고 프리뷰에서는 안 보입니다. 자막 스타일링, 효과음, 속도 조절 등 기본 편집 기능이 부족합니다.

현실적 로드맵

단기 목표로 2-3주 안에 도달 가능한 "괜찮은 유튜브 쇼츠" 수준이 있습니다. 여기에 필요한 것은 로열티프리 BGM 라이브러리 내장, TTS 나레이션 자동 타이밍, 자막 자동 생성 + 스타일링, 그리고 i2v 클립 연결을 매끄럽게 하는 것입니다.

중기 목표로 1-2개월이면 "인상적인 AI 단편" 수준에 도달할 수 있습니다. 더 나은 비디오 모델 통합 (HunyuanVideo 등), 캐릭터 일관성 (IPAdapter + 레퍼런스 시트), 립싱크 통합이 필요합니다.

장기적으로 보면, Sora/Kling/Runway 같은 상용 서비스 수준은 현재 로컬 GPU로는 도달이 어렵습니다.

가장 임팩트 있는 다음 3가지

첫째, 로열티프리 BGM 파일을 10-20곡 내장하는 것이 ACE-Step 합성보다 100배 낫습니다. 둘째, TTS 자동 싱크로 나레이션 길이에 맞춰 클립 타이밍을 자동 조정하는 것입니다. 셋째, 자막 자동 생성으로 TTS 텍스트를 타임코드 기반 자막으로 변환하는 것입니다.

이 세 가지만 완성하면 "나레이션 + 자막 + 배경음악 + AI 영상"이 갖춰진 쇼츠 콘텐츠를 만들 수 있습니다. GPU 성능에 관계없이 가능한 작업들이라 가성비가 가장 높습니다.

현재 GPU 사양이 어떻게 되나요? 그에 따라 비디오 모델 전략이 달라집니다.

이미 문서 어디에 다 기록되어있는걸 다시 엉뚱한 판단으로 모델을 바꾸면 안됩니다
죄송합니다. 기존 문서와 파이프라인을 무시하고 판단한 부분이 잘못되었습니다.

현재 파이프라인(Wan2.2 i2v + 업스케일 + ACE-Step)은 이미 결정된 사항이고, 모델을 바꾸는 것이 아니라 현재 파이프라인 안에서 품질을 최대로 끌어올리는 것이 맞습니다.

말씀하신 대로 임팩트 있는 다음 작업 3가지로 진행합니다:

TTS 자동 싱크 — 나레이션 길이에 맞춰 클립 타이밍 자동 조정
자막 자동 생성 — TTS 텍스트를 타임코드 기반 자막으로 변환
BGM 개선 — 로열티프리 BGM 파일 내장 또는 ACE-Step 품질 개선
기존 문서에 정리된 로드맵과 우선순위를 다시 확인하고 싶습니다. 프로젝트 문서 경로를 알려 주시면 먼저 읽고 그에 맞춰 진행하겠습니다.

========== data-flow.md ==========
# FlowCut Data Flow

## Overview

FlowCut has two primary data flows:
1. **Manual editing** — user imports media, places clips on timeline, exports
2. **AI pipeline** — user types prompt, system generates everything automatically

## Manual Editing Flow

```
User drops media → mediaResolver → editorStore.addMediaItem()
                                  → clipFactory.createClip()
                                  → editorStore.addClip()
User edits timeline → Command pattern → editorStore (Zustand)
User exports → /api/export → FFmpeg → output MP4/WebM/GIF
```

## AI Pipeline Flow

```
1. User prompt
   └→ AIBridge.generateScenes(prompt)
      └→ Ollama LLM → 15 scene descriptions

2. Scene planning
   └→ VideoDirector.buildDirectorPlan(scenes, opts)
      └→ scenicTemplate() → DirectorPlan (beats with timing)
      └→ planToFlowScript() → FlowScript JSON

3. Script execution (ScriptEngine.execute)
   ├→ setupProject() → set dimensions, fps from script or DEFAULT_PROJECT
   ├→ importMedia() → for each media item:
   │   ├→ POST /api/comfyui/generate-image → ComfyUI → PNG
   │   └→ POST /api/comfyui/generate-video → ComfyUI Wan2.2 i2v → VHS → MP4
   │       └→ mediaIdMap: original_id → original_id_video
   ├→ createTracks() → video + text tracks
   ├→ createClips() → place clips on timeline
   │   └→ auto-reclassify: _video mediaId or .mp4 src → type: "video"
   └→ executeActions() → auto-subtitle, export

4. Export
   └→ POST /api/export
      └→ FFmpeg command:
          ├→ black background (project dimensions, duration, fps)
          ├→ video inputs (MP4 from i2v, no -loop)
          ├→ image inputs (PNG with -loop 1)
          ├→ filter_complex: scale, pad, overlay, zoompan (Ken Burns)
          ├→ text overlays (pre-rendered PNG)
          └→ output MP4 (H.264)
```

## Project Save/Load Flow

```
Save: editorStore → ProjectManager.saveProjectToFile/LocalStorage()
      → serialize clips, tracks, media, project settings → JSON

Load: JSON → ProjectManager.restoreProject()
      → restore media items
      → restore clips (auto-detect video: _video suffix or .mp4 extension)
      → restore tracks, project settings
```

## Key Constants

- DEFAULT_PROJECT (src/types/project.ts): width 1920, height 1080, fps 30
- Value priority: FlowScript specified > editorStore > DEFAULT_PROJECT
- See CODING_STANDARDS.md for rules

# FlowScript API Reference

FlowCut의 모든 기능은 FlowScript JSON을 통해 스크립트로 실행 가능해야 합니다.
이 문서는 현재 구현 상태와 미구현 액션을 정의합니다.

## 설계 원칙

1. **모든 기능은 스크립트로 실행 가능** — UI 전용 기능 금지
2. **AI가 자유롭게 조합 가능** — 액션 간 의존성 최소화
3. **값 우선순위** — 스크립트 지정값 > editorStore > DEFAULT_PROJECT
4. **멱등성** — 같은 스크립트를 두 번 실행해도 동일한 결과

# FlowCut World Context v2.0

## 0. Open Architecture

FlowCut은 특정 AI 엔진/하드웨어/클라우드에 종속되지 않습니다.
모든 AI 기능은 Provider Interface로 추상화됩니다.

| 기능 | 로컬 옵션 | 클라우드 옵션 |
|------|----------|-------------|
| 이미지 | ComfyUI (Flux/SDXL) | fal.ai, Replicate |
| 비디오 | ComfyUI (Wan2.2/LTX) | fal.ai, Runway, Kling |
| TTS | Edge TTS, XTTS-v2, Fish Speech | ElevenLabs |
| 보간 | ComfyUI RIFE/GMFSS | FFmpeg minterpolate |
| LLM | Ollama | OpenAI, Anthropic |
| 업스케일 | ComfyUI RealESRGAN | FFmpeg lanczos |

## 1. 문제: 영상 일관성 비용

전통 제작: 의상/소품 기록 수백장, 재촬영 하루 $50K+

## 2. World Context 설계

FlowScript에 world 섹션 추가. 한번 정의하면 모든 씬에 자동 적용.

원칙:
1. 최소 노력 - LLM이 자동 생성
2. 점진적 복잡도 - 없으면 기존처럼 동작
3. 참조 이미지 우선 - 없으면 텍스트 폴백
4. 렌더러 독립 - Provider 비종속

world 섹션: visualStyle, era, genre, characters, locations, props, motionLibrary, continuity

---

## 3. 캐릭터 레지스트리

시드 고정으로 동일 프롬프트+시드 = 동일 이미지.
seed를 DB 등록하면 캐릭터 키 하나로 호출.
seed+1, seed+2로 정면/측면/전신 시트 자동 생성.
프롬프트 92% 단축 (1250자 -> 100자/5씬).

---

## 4. 목소리 시스템

캐릭터별 voice 설정:
- engine: auto (Provider 설정 따름)
- fallbackChain: [xtts-v2, fish-speech, edge-tts]
- sampleClip: 보이스 클로닝용 6초 샘플 (옵션)
- preset: Edge TTS 프리셋 (기본)

## 5. 모션 라이브러리
